# 技术决策记录 (Architecture Decision Record, ADR)

## ADR-003: FRP 内网穿透与 Caddy 反向代理部署决策

### 1. 部署与外网访问背景
由于本项目采用纯网页模式运行于用户本机，为实现公网安全访问，需通过内网穿透技术将本机的 `20260` 端口暴露至公网。用户已配置域名 `fifa2026.liukun.com` 指向公网 FRP 服务器 (`47.103.55.200`)。

### 2. 技术选型与拓扑设计
我们决定采用 **FRP 客户端 (frpc) + FRP 服务端 (frps) + Caddy 反向代理** 的组合：
- **本地服务监听**：Go 后端服务默认监听端口固定为 `20260`。
- **内网穿透 (FRP)**：本地运行 `frpc`（FRP 客户端），连接至远程 FRP 服务器 (`frp.syhsgis.com:7000` 或本地 config 对应端口)。在 `frpc.toml` 中配置 HTTP 规则，将本地 `20260` 端口映射到远程服务器的 `vhost_http_port`（通常为远程中转端口）。
- **反向代理与自动 SSL (Caddy)**：公网 FRP 服务器上部署 `Caddy`。修改 `/etc/caddy/Caddyfile`，配置域名反向代理至 FRP 中转端口。由 Caddy 自动申请 Let's Encrypt SSL 证书，提供公网 HTTPS 安全访问。

### 3. Caddy 代理路由设计
Caddyfile 中我们将配置 `fifa2026.liukun.com`，确保公网请求被精准转发至本地的 `20260` 监听端口。
