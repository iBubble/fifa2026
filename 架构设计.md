# 架构设计 - FIFA2026 足球量化分析预测网站

## 1. 物理与通信拓扑 (内网穿透部署)
为使公网安全访问本机运行的 `20260` 服务，系统集成了 FRP 与 Caddy 反向代理。总体网络与部署拓扑如下：

```
                              ┌──────────────────────────────────────────────┐
                              │           公网 FRP 服务器 (47.103.55.200)      │
                              │                                              │
 ┌──────────────────┐  HTTPS  │  ┌───────────┐      SSL      ┌────────────┐  │
 │  外部公网浏览器   ├───────►│  │   Caddy   ├──────────────►│    frps    │  │
 │                  │         │  │ (Port 443)│ (Auto Cert)   │(中转映射口)│  │
 └──────────────────┘         │  └───────────┘               └─────┬──────┘  │
                              └────────────────────────────────────┼─────────┘
                                                                   │ (FRP 隧道连接)
                                                                   ▼
                              ┌──────────────────────────────────────────────┐
                              │                  开发机本机                  │
                              │                                              │
                              │  ┌───────────┐               ┌────────────┐  │
                              │  │    frpc   │◄──────────────┤ Go Backend │  │
                              │  │ (FRP 客户端)              │(Port 20260)│  │
                              │  └───────────┘               └────────────┘  │
                              └──────────────────────────────────────────────┘
```

## 2. 核心模块与实现细节

### 2.1 端口绑定与 Go 服务 (`src/main.go`)
后端 Gin 服务默认绑定本机 `0.0.0.0:20260`（可通过 `.env` 的 `PORT` 变量进行覆盖），保障局域网和本地 `frpc` 顺利转发。

### 2.2 FRP 隧道映射结构
本地 `frpc` 将以 TCP/HTTP 协议连接到中转服务器 `frp.syhsgis.com` 的控制端口（通常为 7000）。`frpc.toml` 配置将本地 `127.0.0.1:20260` 代理到服务器的远程端口（如 `20260` 或是 http 中转端口）。

### 2.3 Caddy 反向代理与证书申请
公网服务器 Caddy 负责在 443 端口监听 `fifa2026.liukun.com` 域名。在其 `/etc/caddy/Caddyfile` 中配置 `reverse_proxy`，将外部请求转发到本地 `frpc` 在服务器侧映射的输出端口，由 Caddy 自动与 Let's Encrypt 完成 ACME 证书申请及续签。
